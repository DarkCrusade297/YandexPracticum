# 📅 Event Manager System
- GET /events - получение списка всех мероприятий
- POST /events - создание мероприятия
- GET /events/{id} - получение мероприятия по его уникальному идентификатору
- PUT /events/{id} - изменение данных о мероприятии по его уникальному идентификатору
- DELETE /events/{id} - удаление мероприятия по его уникальному идентификатору
  
# Метод получения списка всех мероприятий принимает следующие параметры
- title (регистронезависимый поиск по полному или частичному совпадению наименованию мероприятия)
- from (поис мероприятий не раньше указанной даты)
- to (поиск мероприятий не позже указанной даты)
- page (разбитие списка мероприятий на пачки)
- pageSize (количетсво мероприятий в пачке)

# Единый формат ошибки
```
{
  "status": код,
  "detail": "сообщение"
}
```

# Запуск и тестирование проект
- Для того, чтобы выполнить сборку проекта используйте команду "dotnet build"
- Для запуска проекта используйте команду "dotnet run"
- Для проведения тестирования используйте команду "dotnet test"

## 📤 Пример запроса событий с применение фильтров и пагинации
```
  curl -X 'GET' \
    'https://localhost:7100/events?title=1&from=2026-04-27&to=2026-04-29&page=1&pageSize=1' \
    -H 'accept: text/plain'
```
## 📥 Пример ответа
```
  {
    "total": 2,
    "events": [
      {
        "id": "fa632728-de06-4db6-8b6a-09864e9813b1",
        "title": "1",
        "description": "string",
        "startAt": "2026-04-27T19:44:24.551Z",
        "endAt": "2026-04-28T19:44:24.551Z"
      }
    ],
    "currentPage": 1,
    "pageSize": 1
  }
```
### ❌ HTTP 404 Not Found
Возвращается, если не найдено событие по указанному id
```
{
  "status": 404,
  "detail": "Event with id fa632728-de06-4db6-8b6a-09864e9813b0 not found"
}
```
