# 📅 Event Manager System
- GET /events - получение списка всех мероприятий
- POST /events - создание мероприятия
- GET /events/{id} - получение мероприятия по его уникальному идентификатору
- PUT /events/{id} - изменение данных о мероприятии по его уникальному идентификатору
- DELETE /events/{id} - удаление мероприятия по его уникальному идентификатору
  
# Метод получения списка всех мероприятий принимает следующие параметры
- title (регистронезависимый поиск по полному или частичному совпадению наименованию мероприятия)
- from (поис мероприятий не раньше указанной даты)
- to (поиск мероприятий не позже указанной даты)
- page (разбитие списка мероприятий на пачки)
- pageSize (количетсво мероприятий в пачке)

# Единый формат ошибки
{
  "status": код,
  "detail": "сообщение"
}
